算法介绍

算法的定义

算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示一个或者多个操作。

算法的特性

输入输出

算法有 0 个 或 多个 输入；

算法至少有 一个 或 多个 输出。

有穷性

指算法在执行有限的步骤后自动结束，不会无限循环，且每一步骤在可接受的时间完成。

确定性

算法每一步骤都有明确的含义，不会具有二义性。

可行性

算法每一步都是可行的，即，每一步都能够通过执行有限次数完成。

算法设计的要求

“算法的特性”是指：一旦被认为是算法，就具备这些特性；

“算法设计的要求”是指：好的算法应该具备的特点。

正确性

算法正确性是指算法至少有输入输出和无二义性，能正确反应需求，得到正确答案。

算法的“正确”分为 4 个层次：

  1. 算法没有语法错误
  2. 算法对于合法的输入能产出满足要求的输出
  3. 算法对于非法的输入能得到满足规格说明的结果
  4. 算法对于精心选择的刁难的测试数据都有满足要求的输出

一般把 3 作为算法是否正确的基本标准。

可读性

算法设计的另一目的是方便阅读理解和交流。

健壮性

输入数据不合法也能做出相关处理，不是产生异常或莫名其妙的结果。

时间效率高和存储量低

设计算法应该尽量满足时间效率高和存储量低的需求。

算法效率的度量方法

事后统计方法（不推荐）

定义：通过设计好的测试程序和数据，利用计算机计时器对不同算法编制的程序的运行时间进行比较，确定算法效率的高低。

缺陷：

  1. 必须编制好程序才能测试，浪费成本。
  2. 依赖软硬件等环境因素，会产生误差。
  3. 与测试数据的规模有很大关系。

事前分析估算方法

定义：在计算机程序编制前，依据统计方法对算法进行估算。

时间效率取决于：

  1. 算法采取的策略和方法
  2. 编译的代码质量
  3. 问题的输入规模
  4. 机器执行指令的速度

1 是根本，2 取决于软件，4 取决于硬件。因此，程序运行时间依赖于算法的好坏和问题输入的规模。

计算对运行时间有消耗的基本操作的次数可以估计时间效率。

不计循环索引的递增和循环终止条件，最终，分析程序的运行时间最重要的是把程序看成是独立于程序设计语言的算法或一系列步骤。

函数的渐进增长

算法 A：4n + 8

算法 B：n2

给定的两个函数 f(n) 和 g(n)，如果存在一个整数 N, 使得对于所有的 n > N，f(n) 总是比 g(n) 大，那么，我们说 f(n) 的渐进增长快于 g（n）

我们发现，与最高次项相乘的常数并不重要；

最高此项的指数大的，随着 n 增长，结果也会增长得特别快；

判断一个算法的效率时，函数中的常数和其他次要项可以忽略，而更应该关注主项（最高阶项）的阶数。

算法时间复杂度

算法时间复杂度定义

语句总的执行次数 T(n) 是关于问题规模 n 的函数，进而分析 T(n) 随 n 的变化情况确定 T(n) 的数量级。算法的时间复杂度，也就是算法的时间量度，记作：T(n) = O(f(n))

O(1) 叫常数阶，O(n)叫线性阶，O(n2) 叫平方阶

推导大 O 阶方法

  1. 用常数 1 取代运行时间中的所有加法常数。
  2. 在修改后的运行次数函数中，只保留最高阶项。
  3. 如果最高阶项存在且不为 1，去除与这个项相乘的常数。

得到的结果就是大 O 阶。

常数阶

无论常数是多少，都记做 O(1)。

对于分支结构，无论真假，执行的次数都是恒定的，单纯的分支结构的时间复杂度也是 O(1)。

线性阶

分析算法的复杂度，关键是分析循环结构的运行情况。

对数阶

每次 count × 2 后，离 n 更近了一步，2x = n，得到 x = log2n，因此时间复杂度为 O(logn)。

平方阶

循环的时间复杂度等于循环体的时间复杂度乘以该循环运行的次数。

大 O 推导的理解不难，难的是数列的运算，需要数学知识能力。

常见的时间复杂度

O(1) < O(logn) < O(n) < O(nlogn) < O(n2) < O(n3) < O(2n) < O(n!) < O(nn)

最坏情况和平均情况

最坏情况运行时间是一种保证，那就是运行时间不会再坏了。除非特别指定，我们提到的运行时间都是最坏情况的运行时间。

平均运行时间是所有情况中最有意义的，因为它是期望的运行时间。

算法空间复杂度

算法的空间复杂度通过计算算法所需的存储空间实现：S(N) = O(f(n))

有时候以空间换时间的方法可以减少运行时间。

如判断年份是否为闰年，可以实现设定一个数组，数组索引和年份对应，数组值为 0 或 1，闰年为 1，这样就可以减少时间复杂度。但是同时这也增加了内存占用量。
