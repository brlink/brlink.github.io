计算机网络

为什么要分层

发送文件前：

  1. 将数据通信的通路==激活==
  2. 告诉网络如何识别目的主机
  3. 发起主机要查明目的主机是否开机，连接是否正常
  4. 发起主机要知道对方管理程序是否已经做好接收准备
  5. 确保差错和意外可以解决
    …

怎么分层

基本规则

  1. 各层之间相互独立，每层只实现相对独立的功能
  2. 每层之间界面自然清晰，交流尽量少
  3. 结构上可分割。每层用最合适的技术
  4. 下层对上层独立，上层对下层单向提供服务
  5. 整个分层促进标准化

认识分层结构

  1. 实体：第 n 层的活动元素称为 n 层实体。同一层为对等实体。
  2. 协议：对等实体数据交换建立的规则称为网络协议。==水平==
    • 语法：规定传输数据的格式
    • 语义：规定索要完成的功能
    • 同步：规定各种操作的顺序
  3. 接口（访问服务SAP）：上层使用下层服务的入口
  4. 服务：下层为相邻上岑提供的功能调用。==垂直==

SDU 服务数据单元：完成用户所要求的功能而应传送的数据
PCI 协议控制信息：控制协议操作的信息
PDU 协议数据单元：对等层次之间传送的数据单位

$SDU + PCI = PDU$ 然后 PDU 最为下一层的 SDU

总结

  • 网络体系结构从功能上描述计算机网络结构
  • 计网体系简称网络体系结构，是==分层结构==
  • 每层遵循某些==网络协议==完成本层功能
  • 计网体系结构是计网==各层及其协议==的集合
  • n 层为 n+1 层提供服务时，不仅包含第 n 层本身的功能，还包含下层服务提供的功能
  • 相邻层间有接口，且所提供服务的具体实现细节对上层完全屏蔽
  • 体系结构抽象，实现由软硬件实现
